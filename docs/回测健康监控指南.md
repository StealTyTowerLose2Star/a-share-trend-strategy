# 回测健康监控指南

**创建时间**: 2026-04-01  
**版本**: v1.0

---

## 🎯 功能概述

实时监控回测运行状态，发现异常立即通知，避免长时间失败而不知情。

---

## 🔍 监控功能

### 1. 进程监控
- ✅ 检测回测进程是否存活
- ✅ CPU/内存占用监控
- ✅ 进程崩溃检测

### 2. 进度监控
- ✅ 完成/失败股票数统计
- ✅ 失败率检测（阈值 50%）
- ✅ 进度停滞检测（阈值 10 分钟）

### 3. 日志监控
- ✅ 自动扫描错误日志
- ✅ 异常堆栈捕获
- ✅ 关键字匹配（Error/Exception）

### 4. 自动通知
- ✅ 控制台输出
- ✅ 日志文件记录
- ✅ 警报文件保存
- 📱 飞书推送（待配置）

---

## 📊 警报级别

| 级别 | 触发条件 | 处理方式 |
|------|----------|----------|
| **INFO** | 正常信息 | 记录日志 |
| **WARNING** | 进度停滞>10 分钟 | 检查网络和性能 |
| **CRITICAL** | 失败率>50% 或进程崩溃 | 立即介入处理 |
| **SUCCESS** | 回测成功完成 | 查看结果 |

## 🚀 使用方法

### 方式一：持续监控（推荐）


## ⚠️ 常见问题

### Q1: 失败率过高怎么办？

**检查步骤**:
1. 查看日志最后 100 行：`tail -100 logs/backtest_*.log`
2. 检查错误类型：`grep "Error" logs/backtest_*.log`
3. 验证数据源：`python3.10 core/market_scanner.py`

**常见原因**:
- AKShare 接口变化
- 网络连接问题
- 股票代码无效

**解决方法**:
- 更新 AKShare：`pip3.10 install --upgrade akshare`
- 检查网络：`ping akshare.akfamily.xyz`
- 过滤无效代码

---

### Q2: 进度停滞怎么办？

**检查步骤**:
1. 查看进程状态：`ps aux | grep run_backtest`
2. 检查 CPU 占用：`top -p <PID>`
3. 查看系统负载：`uptime`

**常见原因**:
- 数据下载慢
- 系统资源不足
- 程序卡死

**解决方法**:
- 减少线程数：`--workers 2`
- 增加超时时间
- 重启回测：`rm state/backtest_progress.json && python3.10 scripts/run_backtest_parallel.py --reset`

---

### Q3: 如何查看实时进度？

```bash

print(f'失败：{len(d[\"failed\"])} 只')
"

# 方法三：查看日志
tail -f logs/backtest_*.log
```
