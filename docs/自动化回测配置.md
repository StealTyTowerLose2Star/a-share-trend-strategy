# 自动化回测配置指南

## 📋 配置步骤

### 1. 设置脚本执行权限

```bash
cd /home/admin/.openclaw/workspace/a-share-trend-strategy

# 设置执行权限
chmod +x scripts/daily_backtest.sh
chmod +x scripts/run_backtest_parallel.py
```

---

### 2. 配置 Crontab 定时任务

```bash
# 编辑 crontab
crontab -e

# 添加以下行（每日 00:00 运行）
0 0 * * * /home/admin/.openclaw/workspace/a-share-trend-strategy/scripts/daily_backtest.sh

# 保存退出
```

**验证 crontab**：
```bash
# 查看已配置的任务
crontab -l

# 应该看到:
# 0 0 * * * /home/admin/.openclaw/workspace/a-share-trend-strategy/scripts/daily_backtest.sh
```

---

### 3. 测试运行

#### 首次运行（手动测试）

```bash
cd /home/admin/.openclaw/workspace/a-share-trend-strategy

# 运行回测（4 线程）
python3.10 scripts/run_backtest_parallel.py --workers 4
```

**预计时间**：2.5-3.5 小时

**输出**：
- 进度显示（每 50 只股票）
- 日志文件：`logs/backtest_YYYYMMDD.log`
- 结果文件：`backtest/results/daily/backtest_YYYYMMDD.json`

---

### 4. 查看结果

#### 查看最新回测结果

```bash
# 查看最新结果摘要
cat backtest/results/daily/backtest_$(date +%Y%m%d).json | python3.10 -c "
import sys, json
data = json.load(sys.stdin)
print(f'回测股票：{data[\"total_stocks\"]} 只')
print(f'平均收益：{data[\"avg_return\"]}%')
print(f'盈利股票：{data[\"profitable_stocks\"]} ({data[\"profitable_stocks\"]/data[\"total_stocks\"]*100:.1f}%)')
print(f'平均胜率：{data[\"avg_win_rate\"]}%')
"
```

#### 查看日志

```bash
# 查看今日日志
tail -f logs/backtest_$(date +%Y%m%d).log

# 查看历史日志
ls -la logs/
```

## ⚙️ 配置选项

### 调整线程数

**根据 CPU 核心数调整**：

```bash
# 查看 CPU 核心数
nproc

# 推荐配置:
# 4 核：--workers 2
# 8 核：--workers 4
# 16 核：--workers 8
```

**修改 crontab**：
```bash
crontab -e

# 修改为 8 线程
0 0 * * * /home/admin/.openclaw/workspace/a-share-trend-strategy/scripts/daily_backtest.sh --workers 8
```

### 调整回测周期

**修改回测脚本**：
```python
# scripts/run_backtest_parallel.py
start_date = '2019-01-01'  # 修改开始日期
end_date = '2025-12-31'    # 修改结束日期
```

### 重置进度（重新跑）

```bash
# 清除进度文件，重新跑全量
python3.10 scripts/run_backtest_parallel.py --reset --workers 4
```

## 📊 输出文件

### 日志文件

**位置**: `logs/backtest_YYYYMMDD.log`

**内容**:
```
========================================
启动时间：2026-04-01 00:00:00
回测任务启动
========================================

开始回测...
进度：50/4500 (1.1%)
  已完成：1250
  失败：15
  已用时间：30.5 分钟
  预计剩余：150.2 分钟

========================================
完成时间：2026-04-01 03:15:00
✅ 回测成功完成
========================================
```

### 结果文件

**位置**: `backtest/results/daily/backtest_YYYYMMDD.json`

**结构**:
```json
{
  "backtest_date": "2026-04-01 03:15",
  "start_date": "2019-01-01",
  "end_date": "2025-12-31",
  "total_stocks": 4500,
  "avg_return": 125.5,
  "median_return": 98.3,
  "profitable_stocks": 3200,
  "avg_win_rate": 62.5,
  "avg_max_drawdown": -15.8,
  "results": [...]
}
```

### 进度文件

**位置**: `state/backtest_progress.json`

**作用**: 断点续跑，避免重复计算

**重置**:
```bash
rm state/backtest_progress.json
```

## 🔍 监控与维护

### 监控运行状态

```bash

ps aux | grep run_backtest

```

### 故障处理

#### 回测中断

```bash
# 直接重新运行（会自动续跑）
python3.10 scripts/run_backtest_parallel.py --workers 4
```

#### 数据错误

```bash
# 重置进度，重新跑
python3.10 scripts/run_backtest_parallel.py --reset --workers 4
```

#### 日志过大

```bash
# 清理 30 天前的日志
find logs/ -name "*.log" -mtime +30 -delete
```

## ⚠️ 注意事项

### 资源占用

- **CPU**: 4 线程约占用 50% CPU
- **内存**: 约 2-4GB
- **磁盘**: 缓存数据约 10-20GB
- **网络**: 下载数据约 1-2GB/次

### 运行时间

- **4 线程**: 2.5-3.5 小时
- **8 线程**: 1.5-2 小时
- **建议**: 凌晨 00:00 运行，早上 03:00 前完成

### 数据更新

- **每日运行**: 获取最新数据
- **周末**: 可跑全量回测
- **月初**: 建议重置重新跑
